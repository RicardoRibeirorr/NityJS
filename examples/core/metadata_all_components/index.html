üîß Metadata Creation Methods

// Method 1: Traditional Constructor
const sprite = new SpriteRendererComponent("player", {
    width: 64,
    height: 64,
    opacity: 0.8
});

// Method 2: Metadata Factory
const sprite = SpriteRendererComponent.meta({
    spriteName: "player",
    width: 64,
    height: 64,
    opacity: 0.8
});

// Method 3: Component.createFromMetadata
const sprite = Component.createFromMetadata(SpriteRendererComponent, {
    spriteName: "player",
    width: 64,
    height: 64,
    opacity: 0.8
});

üìä Metadata Features

  • Default Values: Every component provides default metadata via getDefaultMeta()
  • Validation: All metadata is validated with helpful error messages
  • JSON Serialization: Components can be serialized to/from JSON for visual editors
  • Backward Compatibility: Traditional constructor usage still works perfectly
  • Type Safety: Validation ensures correct data types and value ranges
  • Unity Familiarity: Metadata design follows Unity Inspector patterns

üîç Validation Examples

The system validates all metadata and provides helpful error messages:

// ‚ùå This will throw an error

    opacity: 2.0  // Must be between 0 and 1
});

// ‚ùå This will throw an error  
BoxColliderComponent.meta({
    width: -10  // Must be positive
});

// ‚úÖ This will work perfectly
SpriteRendererComponent.meta({
    spriteName: "player",
    opacity: 0.8,
    flipX: true
});
